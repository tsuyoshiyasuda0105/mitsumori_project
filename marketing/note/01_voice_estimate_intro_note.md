## 建設業は、もう「頑張れば回る」だけではきつい

建設業は、昔から忙しい業界です。

現場は天候に左右されます。  
人の段取りもあります。  
材料も動きます。  
お客さんへの説明もあります。

それに加えて、いまは働き方のルールも変わっています。

厚生労働省の案内では、2024年4月1日から建設業にも時間外労働の上限規制が適用されています。  
つまり、長く働いて帳尻を合わせるやり方は、だんだん取りづらくなっています。

一方で、国土交通省の資料では、建設業は他産業と比べて労働生産性が低い状況にあること、そして就業者の高齢化が進んでいることも示されています。  
建設業就業者は55歳以上が36.7％、29歳以下が11.7％という数字も出ています。

これは「人が足りないから大変」というだけの話ではありません。

人が少なくなる。  
働ける時間にも限りがある。  
でも仕事の品質は落とせない。

この状態で必要になるのは、根性論ではなく、仕事の流れを少しずつ軽くすることだと思います。

| 建設業で起きている変化 | 現場への影響 |
| --- | --- |
| 時間外労働の上限規制 | 長時間労働で吸収しにくくなる |
| 高齢化 | ベテランの経験をどう残すかが課題になる |
| 若手不足 | 教育と引き継ぎに時間がかかる |
| 原価・材料費の変動 | 見積の精度とスピードがより重要になる |
| システム化の遅れ | 二重入力や手戻りが残りやすい |

大きな改革をいきなりやるのは難しいです。  
でも、毎日残っている小さな無駄を減らすことなら始められます。

## 生産性を上げるとは、現場の人を急がせることではない

生産性向上という言葉には、少し怖さもあります。

「もっと早くやれ」  
「もっと件数をこなせ」  
「同じ人数で売上を増やせ」

そんなふうに聞こえることがあるからです。

でも、建設業でそれをやると無理が出ます。  
現場の確認を雑にすれば、あとで手戻りが起きます。  
見積を急ぎすぎれば、抜けや金額ミスが起きます。

だから、現場の人を急がせる方向ではなく、**現場の人が本来やらなくていい作業を減らす方向**で考えたい。

たとえば、こんな作業です。

| 減らしたい作業 | 理由 |
| --- | --- |
| 同じ内容の二重入力 | 見積、施工管理、会計で同じ情報を打つのは重い |
| 単価表の探し直し | 毎回Excelや過去見積を見る時間が積み上がる |
| メモの読み返し | 現場で話した内容が後から曖昧になる |
| 担当者ごとの価格判断 | 会社としての価格基準がぶれやすい |
| 見積作成の後回し | 商談の温度が下がる |

こういう作業は、仕事の価値そのものではありません。

お客さんに説明する。  
現場を正しく見る。  
必要な工事を判断する。  
安全に施工する。

こちらが本当に大事な仕事です。

生産性を上げるというのは、現場の人から大事な仕事を奪うことではなく、大事な仕事に時間を戻すことなんだと思います。

## 見積作成は、建設業の生産性にかなり効く

建設業の業務改善というと、施工管理アプリや現場写真管理、日報、勤怠などがよく出てきます。

もちろん、それも大事です。

ただ、私は「見積作成」もかなり重要だと思っています。

理由はシンプルです。  
見積は、売上の入口だからです。

見積が遅れると、受注のタイミングを逃すことがあります。  
見積の単価がぶれると、利益もぶれます。  
見積明細が整理されていないと、その後の施工管理や請求にも影響します。

つまり見積は、ただの書類ではありません。

営業、原価、施工、請求、会計。  
その全部につながる最初のデータです。

| 見積で整えること | 後工程への影響 |
| --- | --- |
| 品目 | 施工内容の整理につながる |
| 数量 | 原価・材料・職人手配に影響する |
| 単位 | 比較や集計がしやすくなる |
| 単価 | 利益管理に影響する |
| 外部ID | 他システムへの連携に使える |
| 備考 | 顧客説明や社内共有に役立つ |

見積作成を整えることは、会社全体の仕事の流れを整えることに近いです。

だからこそ、見積を「その場限りのExcel」だけで終わらせるのではなく、後工程にも使えるデータにしておく価値があります。

## AIは現場の代わりではなく、下書きを作る道具でいい

AIという言葉が出ると、どうしても大げさに聞こえます。

「AIが全部自動でやります」  
「人はいりません」  
「見積も自動で完成します」

こういう言い方は、建設業には合わないと思っています。

現場ごとの判断があります。  
お客さんとの話し方があります。  
会社ごとの単価や施工基準もあります。

そこをAIが勝手に決めるのは危ないです。

だから、ボイス見積ではAIを「見積明細の下書きを作る道具」として考えています。

現場で話した内容をもとに、品目や数量、備考の候補を作る。  
単位や単価は、会社の単価マスターから引く。  
最後は人が確認する。

このくらいの距離感が、現実的だと思います。

| 項目 | AIに任せる | 人が確認する |
| --- | --- | --- |
| 現場の音声から明細候補を作る | はい | はい |
| 品目を単価マスターから推測する | はい | はい |
| 数量を候補として出す | はい | はい |
| 単位を自由に決める | いいえ | 単価マスターを使う |
| 単価を自由に決める | いいえ | 単価マスターを使う |
| 最終見積を確定する | いいえ | はい |

AIは責任者ではなく、下書き担当。  
この考え方なら、現場の判断を残しながら、事務作業だけを軽くできます。

## 単価マスターは、会社の経験を残す場所になる

単価表は、ただの金額表ではありません。

そこには、会社の経験が入っています。

この工事はこの単位で見る。  
この仕様ならこの単価を基準にする。  
この品目はこの外部IDで管理する。

そういう積み重ねです。

でも、Excel単価表が担当者ごとに分かれていたり、過去見積からその場で探していたりすると、会社の経験が整理されにくくなります。

単価マスターを作る意味は、価格を固定することだけではありません。  
会社としての基準を残すことです。

| 単価マスターで持ちたい情報 | 使い道 |
| --- | --- |
| 品目名 | 音声からの品目候補に使う |
| 単位 | 見積明細の単位を統一する |
| 単価 | 担当者ごとの価格ぶれを減らす |
| 外部ID | 他システムへの連携に使う |
| 備考 | 仕様や注意点を残す |

特に外部IDは、今後かなり大事になると思っています。

見積で作ったデータを、施工管理、販売管理、会計などに渡すとき、品目がバラバラだと連携しづらいです。  
でも、単価マスター側にIDを持たせておけば、見積データを次のシステムへ渡しやすくなります。

見積を作って終わりではなく、見積から会社のデータが始まる。  
そう考えると、単価マスターはかなり重要です。

## よくある質問

### Q. AIで見積を作るのは危なくないですか？

AIに全部任せるなら危ないと思います。  
だから、ボイス見積ではAIを下書き担当として使い、最後は人が確認する流れを前提にしています。

### Q. 生産性向上というと、人を減らす話に聞こえます。

このサービスで考えている生産性向上は、人を減らすことではありません。  
二重入力、単価探し、メモの読み返しなど、現場の人が本来やらなくていい作業を減らすことです。

### Q. Excel単価表を使っている会社でも使えますか？

使えるようにする方針です。  
既存のExcel単価表を単価マスターとして取り込み、品目、単位、単価、外部IDを整理する流れを想定しています。

### Q. 小規模な会社にも必要ですか？

むしろ小規模な会社ほど、見積作成の負担が代表や営業担当に集中しやすいです。  
大きなシステムを入れる前に、まず見積作成の流れを軽くする価値はあると思います。

### Q. どの業種に向いていますか？

外壁塗装、屋根工事、リフォーム、設備工事など、現場確認から明細型の見積を作る業種を想定しています。  
単価マスターを持っている会社とは特に相性がよいです。
