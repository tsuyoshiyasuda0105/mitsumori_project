# 先行公開 広報プラン ― ボイス見積

> 前提：本ツールは先行公開中のプロトタイプ（試作）。本フェーズの目的は「正式提供前のフィードバック収集」と「先行利用者（モニター）の獲得」。実績数値はすべて目標／想定であり、確定値ではありません。

---

## 1. 本フェーズのゴール

- **一次目標**：現場で実際に使ってくれる**先行利用者（モニター）[N]者**の獲得を目指す。
- **二次目標**：デモを触ってもらい、**情報分離の安心**という価値が伝わるかを検証する。
- **三次目標**：現場目線のフィードバックを集め、正式版の改善・優先順位づけに反映する。

> ※「導入○社」「○時間削減」などの確定実績は、検証後にあらためて公表する。本フェーズでは断定しない。

---

## 2. ターゲット

| 区分 | 具体像 | 重視する訴求 |
| --- | --- | --- |
| メイン | 外壁塗装・屋根塗装の中小事業者／一人親方（従業員 1〜10名規模） | 現場完結のスピード、内部情報が出ない安心 |
| サブ | リフォーム・内装・防水・外構の小規模事業者 | 単価表の取込、PDF/Excel出力の手軽さ |
| 関与者 | 見積を作る事務担当・奥さま（家族経営の事務方） | 消し忘れ防止、入力の手間削減 |

## 3. チャネル

| チャネル | 役割 | 主なコンテンツ |
| --- | --- | --- |
| ランディングページ（LP） | 受け皿・デモ導線 | 機能説明、情報分離の図解、デモCTA |

| 業界コミュニティ／知人紹介 | 一次モニター獲得 | 塗装・リフォーム関連の交流会、知人事業者への直接案内 |
| プレスリリース配信 | 信頼づけ・第三者露出 | `press_release.md`（配信サービス／業界メディア） |

## 4. 4週間スケジュール

### 第1週：土台づくり（準備・静かな立ち上げ）
| 項目 | 内容 |
| --- | --- |
| LP最終確認 | デモ導線・「試作版」表記・問い合わせ先の確認 |
| 計測準備 | デモ訪問・体験数を測れる状態に（[計測ツール] / [プレースホルダ]） |
| 素材準備 | SNS画像（画面イメージ・情報分離の色分け図）、固定ポスト作成 |
| プレスリリース | 文面確定、配信先リストアップ |
| ソフト公開 | 知人・近しい事業者数名にだけ先行案内し、初期フィードバックを取得 |

### 第2週：公開・認知づくり
| 項目 | 内容 |
| --- | --- |
| プレスリリース配信 | 配信サービス／業界メディアへ |
| SNS本格始動 | X：課題共感型(X-1)で開始、隔日投稿。IG-1・FB-1を投稿 |
| アカウント整備 | プロフィール・固定ポストにデモURLを設置 |
| 反応の一次集計 | どの訴求（速さ／安心／単価取込）が刺さるか観察 |

### 第3週：直接アプローチ・モニター募集
| 項目 | 内容 |
| --- | --- |
| 営業メール／DM | ターゲットリストへ案1・案2を送信（小ロットでABテスト） |
| SNS継続 | 差別化型(X-2)・IG-2・FB-2で情報分離を強めに訴求 |
| モニター募集 | 「先行利用者募集」の告知を投稿・LPに導線追加 |
| 個別対応 | 反応者へ15分のオンライン説明を実施 |

### 第4週：フィードバック集約・次への接続
| 項目 | 内容 |
| --- | --- |
| 声の収集 | デモ体験者・モニターへ簡単なヒアリング（[アンケートURL]） |
| 改善整理 | いただいた声を「すぐ直す／正式版で対応」に仕分け |
| 成果まとめ | KPIの中間レビュー、効いた訴求・チャネルの特定 |
| 発信 | 「いただいた声」を題材にした投稿で継続接点づくり |
| 次フェーズ計画 | 正式版・料金提示に向けたロードマップ草案 |

### 補足
- 数値は仮目標。第1〜2週の実測を見て現実的な水準へ調整する。
- 本フェーズは「売上」より**学習量（フィードバック）とモニター獲得**を重視する。
- 効いた訴求（速さ／安心／単価取込）とチャネルを見極め、次フェーズの広告・営業に集中投下する。

---

## 6. 運用上の注意（広報の前提）

- 「試作・先行公開」である事実を必ず明示する（誇大・虚偽の禁止）。
- 実績・効果は「目指す／想定」または[プレースホルダ]で表現する。
- どの媒体でも**情報分離（顧客に原価・指示が出ない安心）**に一度は触れる。
- いただいた不具合・要望は記録し、改善サイクルに必ず戻す。
