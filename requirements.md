## 2. 背景

現場系の中小事業者では、見積作成において以下の課題がある。

- 現場確認後、事務所に戻ってから見積書を作成しており、対応に時間がかかる。
- 手書きメモ、LINE、電話、Excelなどに情報が分散している。
- 見積項目や単価が担当者ごとにばらつきやすい。
- 既存の見積書フォーマットや単価表がExcelで管理されている。
- 見積作成、請求書作成、顧客対応の事務作業が属人化している。
- AIを活用したいが、既存業務を大きく変えるシステムは導入しづらい。

本システムは、現場で音声入力した内容をAIが構造化し、登録済みの単価マスターをもとに見積項目を生成することで、見積作成時間の短縮、単価の標準化、見積品質の安定化を実現する。

## 3. 目的

本システムの目的は以下の通り。

- スマートフォンから音声で見積内容を入力できるようにする。
- AIにより、音声内容から顧客情報、場所、品目、数量、単位、備考、業者指示事項を抽出する。
- 自社単価マスターをもとに見積明細を自動生成する。
- 見積担当者が確認・修正したうえで、ExcelおよびPDF形式で見積書を出力できるようにする。
- 既存のExcel単価表を取り込めるようにし、導入負荷を下げる。
- 将来的にGoogleスプレッドシート連携、請求書作成、案件管理、過去見積活用へ拡張できる基盤を作る。

## 4. 対象ユーザー

### 4.1 主な利用者

- 経営者、管理者

### 4.2 想定業種

初期ターゲットは、現場確認を伴う中小事業者とする。

- 外壁塗装業
- 内装工事業

MVPでは、業種特化テンプレートを用意しやすい「リフォーム・外壁塗装系」を優先候補とする。

## 5. システム概要

本システムは、スマートフォンおよびPCブラウザから利用できるWebアプリケーションである。

現場担当者はスマートフォンで音声入力を行い、AIが入力内容を解析する。解析結果は見積明細候補として表示され、単価マスターと照合される。担当者または事務担当者は、見積編集画面で内容を確認・修正し、ExcelまたはPDFで見積書を出力する。

## 7. 業務要件

### 7.1 現状業務

1. 現場で顧客から要望をヒアリングする。
2. 担当者が手書きメモ、スマホメモ、LINEなどに内容を記録する。
3. 事務所または帰社後にExcelで見積書を作成する。
4. 単価表や過去見積を確認しながら金額を入力する。
5. PDF化してメールまたはLINEで顧客へ送付する。
6. 必要に応じて修正見積を作成する。

### 7.2 システム導入後の業務

1. 現場でスマートフォンから音声入力する。
2. AIが音声を文字起こしする。
3. AIが作業内容、数量、単位、顧客情報、確認不足項目を抽出する。
4. システムが単価マスターと照合し、見積明細候補を作成する。
5. 担当者が見積内容を確認・修正する。
6. ExcelまたはPDFで見積書を出力する。
7. 必要に応じて顧客へ送付する。

## 11. 外部連携要件

### 11.1 AI音声文字起こし

- 音声データを文字起こしAPIに送信する。
- 日本語音声に対応する。
- 専門用語の誤認識を考慮し、ユーザーが修正できる画面を用意する。

### 11.2 AIテキスト解析

- 文字起こし結果から見積条件を抽出する。
- 抽出結果はJSON形式などの構造化データとして扱う。
- 単価や金額の最終決定はシステムの単価マスターとユーザー確認に基づく。

### 11.3 Excel出力

- システム内の見積データからExcelファイルを生成する。
- 初期は標準テンプレートを提供する。
- 将来的に顧客ごとの独自テンプレート対応を検討する。

### 11.4 PDF出力

- 見積データからPDFファイルを生成する。
- スマートフォンで共有しやすいファイルサイズにする。

### 11.5 将来連携

- Googleスプレッドシート出力
- LINE共有またはLINE Bot
- メール送信
- 会計ソフト連携
- クラウドストレージ連携

| 対応端末 | スマートフォン、タブレット、PC |
| 対応ブラウザ | Chrome、Safari、Edgeの最新版 |
| 画面対応 | レスポンシブ対応 |
| 推奨利用 | スマートフォンで現場入力、PCで管理・編集 |

### 12.2 性能要件

| 項目 | 要件 |
| --- | --- |
| 画面表示 | 通常画面は3秒以内を目標とする。 |
| 音声文字起こし | 1分以内の音声は30秒以内の処理を目標とする。 |
| AI解析 | 通常の見積入力は30秒以内の解析を目標とする。 |
| Excel出力 | 100明細以内の見積は10秒以内の出力を目標とする。 |
| PDF出力 | 100明細以内の見積は10秒以内の出力を目標とする。 |

### 12.3 セキュリティ要件

- 通信はHTTPSで暗号化する。
- パスワードはハッシュ化して保存する。
- 会社ごとにデータを分離し、すべての業務データに会社IDを持たせる。
- 他社の顧客、見積、見積明細、単価マスター、音声、打ち合わせ録音、Excel、PDF、AI解析履歴を参照・更新・削除できないようにする。
- クライアントから送信された会社IDは信用せず、ログインユーザーのセッションから会社IDを確定する。
- DB設計では、会社IDを含む複合外部キーまたはRow Level Security相当の仕組みにより、会社をまたいだ紐づけを防止する。
- 管理者と一般担当者で権限を分ける。
- 音声データ、文字起こしデータ、AI解析データの保存期間を定義する。
- AI APIに送信するデータの取り扱いを利用規約、プライバシーポリシーに明記する。

### 12.4 可用性要件

- 業務時間中に安定して利用できること。
- 障害発生時にデータ消失を防ぐため、定期バックアップを行う。
- 重要データの削除には確認操作を設ける。

### 12.5 保守性要件

- 業種別テンプレートを追加しやすい設計とする。
- Excelテンプレートを将来的に差し替え可能な設計とする。
- AIプロンプトや解析ルールを調整しやすい設計とする。

### 12.6 拡張性要件

- Googleスプレッドシート連携を追加できること。
- 請求書作成機能を追加できること。
- LINE連携を追加できること。
- 業種別テンプレートを追加できること。
- 過去見積データを活用した類似案件提案へ拡張できること。

## 13. 帳票要件

### 13.1 見積書Excel

見積書Excelには以下を表示する。

- 自社名
- 自社住所
- 自社電話番号
- ロゴ
- 見積番号
- 見積日
- 有効期限
- 顧客名
- 件名
- 見積明細
- 業者指示事項
- 小計
- 消費税
- 合計金額
- 備考
- 支払条件

### 13.2 見積書PDF

PDFは顧客提出用の見積書として、会社情報、顧客情報、見積明細、合計金額、備考を表示する。業者指示事項は内部情報のためPDFには表示しない。スマートフォンから共有しやすいよう、A4縦を基本とする。

## 14. AI要件

### 14.1 AIに任せる範囲

- 音声文字起こし
- 入力内容の要約
- 顧客情報の抽出
- 作業内容の抽出
- 見積明細候補の生成
- 音声内容からの単価マスター品目候補推測
- 不足情報、確認事項の提示
- 業者指示事項候補の抽出
- 顧客向け説明文の下書き生成

### 14.2 AIに任せない範囲

- 最終見積金額の確定
- 契約条件の判断
- 保証内容の確定
- 法的判断
- 顧客への自動送信

### 14.3 AI解析時の注意

- AIの結果は候補として扱い、必ずユーザーが確認する。
- 金額は原則として単価マスターを優先する。
- AIが不明確な内容を補完した場合は、補完内容をユーザーに明示する。

## 16. 運用要件

- 管理者は会社情報、単価マスター、ユーザーを管理する。
- 一般担当者は見積作成と顧客登録を行う。
- 単価マスターは定期的に見直す。
- Excel取り込み時は、取り込み前にプレビュー確認を必須とする。
- AI解析結果は必ず人間が確認してから見積に反映する。

## 17. 移行要件

初期導入時には、既存Excel単価表から単価マスターを取り込む。

移行手順は以下とする。

1. 既存Excel単価表を準備する。
2. システムの取り込み画面からExcelをアップロードする。
3. 列マッピングを確認する。
4. エラー行、重複行を確認する。
5. 取り込みを実行する。
6. 単価マスター一覧で登録結果を確認する。

## 18. 競合との差別化要件

既存の簡易見積アプリとの差別化として、以下を重視する。

- LINEだけで完結する簡易PDF作成ではなく、業務用の見積管理に対応する。
- 既存Excel単価表を取り込める。
- 自社単価マスターを優先して金額を計算する。
- 会社独自の見積書Excelフォーマットに将来対応する。
- スマートフォン現場入力とPC事務管理を両立する。
- AIによる聞き漏れチェックを行う。
- 過去見積の再利用、類似案件提案へ拡張できる。

## 19. 成功指標

MVPの成功指標は以下とする。

- 見積作成時間を従来比50%以上削減できる。
- 音声入力から見積明細候補生成までを1分以内に完了できる。
- Excel単価表の初回取り込みを30分以内に完了できる。
- ユーザーがAI解析結果を確認・修正したうえで見積書を出力できる。
- 現場担当者がスマートフォンだけで見積作成の一次入力を完了できる。
