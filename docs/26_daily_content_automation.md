# note・ブログ自動ドラフト生成設計

## 1. 目的

ボイス見積の認知獲得、SEO流入、note経由の共感獲得を継続するため、AIで毎日1本の未公開ドラフトを作成する。

公開は自動化しない。AIはドラフト作成までを担当し、公開判断、事実確認、表現調整は人が行う。

## 2. 運用方針

| 項目 | 方針 |
| --- | --- |
| 生成頻度 | 1日1本 |
| 生成時間 | 毎朝9時を初期値 |
| 生成物 | noteまたはSEOブログのMarkdownドラフト |
| 公開 | 手動 |
| 保存先 | `marketing/content-automation/drafts/` |
| 参照テーマ | `marketing/content-automation/topics.md` |
| 参照プロンプト | `marketing/content-automation/daily-draft-prompt.md` |

## 3. なぜ自動公開しないか

- 先行公開中のプロトタイプであり、実装済み/未実装の境界を確認する必要がある
- 建設業、法務、個人情報、AI表現に関する誤解を避ける必要がある
- SEO記事は内部リンク、CTA、公開日、sitemap反映の最終確認が必要
- noteは文体、温度感、動画挿入位置の人間調整が重要

## 4. 媒体の使い分け

| 媒体 | 目的 | 向いている内容 |
| --- | --- | --- |
| SEOブログ | 検索流入、LP送客 | 見積AI、単価マスター、時間短縮、データ連携 |
| note | 共感、開発背景、SNS連携 | なぜ作るか、現場課題、プロトタイプ公開の背景 |

## 5. 公開フロー

### SEOブログ

1. ドラフトを確認する
2. 事実、誇大表現、CTA、内部リンクを修正する
3. `web/lib/blog.ts` の `BLOG_POSTS` 形式に変換する
4. `npm.cmd run build` を実行する
5. Vercelへデプロイする
6. Search ConsoleでURL検査する

### note

1. ドラフトを確認する
2. note向けに改行、見出し、冒頭フックを整える
3. 必要なら15秒デモ動画を挿入する
4. noteへ手動投稿する
5. Xで告知する

## 6. 品質ゲート

| ゲート | 確認 |
| --- | --- |
| 事実 | 実績、導入社数、削減時間を捏造していない |
| 製品状態 | プロトタイプであることと未実装範囲を誤魔化していない |
| AI表現 | AIが金額を自動確定すると書いていない |
| SEO | タイトル、description、H2、内部リンク、CTAがある |
| note | 共感、背景、読後の行動がある |
| 公開 | 人が確認してから公開する |

## 7. 初期スケジュール

| 曜日 | 生成物 |
| --- | --- |
| 月 | SEOブログ |
| 火 | note |
| 水 | SEOブログ |
| 木 | note |
| 金 | SEOブログ |
| 土 | note |
| 日 | backlogの残りが多い媒体 |

## 8. 改善サイクル

- 週1回、Search Consoleのクエリを見てテーマを追加する
- noteの反応が良いテーマはSEOブログ化する
- SEOブログで検索意図が強いテーマはnoteで開発背景を書く
- 問い合わせにつながったテーマはLPやFAQへ反映する
