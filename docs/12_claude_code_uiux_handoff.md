# Claude Code依頼文 UI/UX設計担当

## 依頼概要

音声AI見積作成システムのUI/UX設計のみを担当してください。

本プロジェクトでは、要件定義、DB設計、API設計、AI連携設計、セキュリティ設計は別担当で進めています。Claude Codeでは、以下の範囲に集中してください。

- 画面構成
- 操作導線
- スマートフォン/PCの使いやすさ
- ワイヤーフレーム
- UIコンポーネント設計
- 入力体験
- 音声入力から見積明細作成までのUX
- Excel/PDF出力操作のUX

DB、API、認証、AIモデル選定、会社間データ分離の仕様は変更しないでください。

## プロジェクト概要

本システムは、営業担当者や現場担当者がスマートフォンで音声入力し、AIが見積明細候補を作成するWebアプリです。

主な流れ:

1. 見積編集画面を開く
2. 顧客を選択する
3. 音声で見積明細を入力する
4. AIが文字起こし、明細候補、単価マスター候補、確認事項を提示する
5. ユーザーが候補を確認・修正する
6. 見積明細へ反映する
7. 明細をドラッグアンドドロップで並び替える
8. ExcelまたはPDFを出力する

## 重要な業務要件

### 見積明細項目

見積明細は以下の項目を持ちます。

- 場所
- 品目
- 数量
- 単位
- 単価
- 金額
- 備考
- 業者指示事項

業者指示事項は内部向け情報です。顧客向けPDFには出力しません。ただし、Excel出力では社内確認用または業者指示用として出力できます。

### 単価マスター

単価マスターは以下の項目を持ちます。

- 品目
- 単位
- 単価

音声入力からAIが単価マスターの品目候補を推測し、ユーザーが候補から選択します。選択後、単位と単価が見積明細に反映されます。

### 打ち合わせ録音オプション

打ち合わせ録音はMVP本体ではなく、別オプションです。オプション有効時のみ、営業担当者は顧客との打ち合わせ内容を録音できます。

打ち合わせ録音では以下を扱います。

- 録音開始前の同意確認
- 録音開始/停止/一時保存
- 文字起こし
- 要約
- 顧客要望
- 確認事項
- 見積反映候補

見積への反映は自動確定せず、必ずユーザーが確認してから反映します。

オプション無効の会社では、見積編集画面、顧客詳細画面、ダッシュボードに打ち合わせ録音への導線を表示しません。

## UI/UXで重視すること

### スマートフォン

現場で片手操作しやすいことを重視してください。

- 見積編集画面からすぐ音声入力できる
- 録音ボタンは押しやすく、状態が明確
- 文字起こし結果を修正しやすい
- AI候補を採用/却下しやすい
- 明細の編集が小さい画面でも破綻しない
- ドラッグ操作が難しい場合は、上下移動ボタンも用意する

### PC

事務所での管理・確認作業を重視してください。

- 見積一覧、顧客一覧、単価マスター一覧を見やすくする
- Excel取り込みのプレビューとエラー確認をしやすくする
- 見積明細を表形式で編集しやすくする
- Excel/PDF出力モードを選びやすくする

## 必須画面

以下の画面について、画面構成とワイヤーフレームを作成してください。

| 画面ID | 画面名 | 主な目的 |
| --- | --- | --- |
| S-001 | ログイン画面 | メール・パスワードでログイン |
| S-002 | ダッシュボード | 最近の見積、作業ショートカット |
| S-005 | 見積編集画面 | 顧客選択、音声入力、明細編集 |
| S-004 | AI解析結果確認画面 | 文字起こし、見積候補、確認事項を確認 |
| S-019 | 打ち合わせ録音画面 | 別オプション。顧客打ち合わせの録音、要約、反映候補確認 |
| S-007 | 見積一覧画面 | 見積の検索、状態確認 |
| S-008 | 顧客一覧画面 | 顧客検索、登録、編集 |
| S-010 | 単価マスター一覧画面 | 品目、単位、単価の管理 |
| S-012 | Excel取り込み画面 | 単価マスターExcelのアップロード |
| S-013 | Excel取り込みプレビュー画面 | 取り込み前確認、エラー修正 |
| S-014 | 会社設定画面 | 会社情報、ロゴ、帳票設定 |

特に優先する画面は、`S-005 見積編集画面`、`S-004 AI解析結果確認画面` です。`S-019 打ち合わせ録音画面` は別オプションとして、MVP主要導線とは分けて設計してください。

## 制約

- DB設計、API設計、AIモデル選定は変更しない
- 会社間データ分離や認証仕様は変更しない
- 顧客向けPDFに業者指示事項を表示しない
- AI結果は自動確定せず、必ずユーザー確認を挟む
- スマートフォンでの現場入力を最優先する
- PCでは一覧性と編集効率を重視する
- 見た目だけでなく、実務での入力ミス削減を重視する

## デザインの方向性

業務用SaaSとして、派手なランディングページ風ではなく、実務で繰り返し使いやすい落ち着いたUIにしてください。

- 情報密度は高め
- 操作ボタンは分かりやすく
- 状態表示を明確に
- エラーや確認事項を見落としにくく
- スマートフォンでは入力欄を詰め込みすぎない
- PCでは表形式編集を快適に

## 完了条件

- 主要画面の操作導線が説明できる
- スマートフォンで音声入力から明細反映まで迷わず操作できる
- PCで単価マスター、顧客、見積を効率よく管理できる
- AI候補をそのまま確定せず、確認・修正できるUXになっている
- 業者指示事項が顧客向け出力に混入しないUIになっている
- 実装担当が画面実装に着手できる粒度になっている
