# 音声AI見積作成システム MVP実装計画書

## MVPゴール

MVPでは、現場担当者がスマートフォンで音声またはテキストから見積作成を開始し、AI解析結果を人間が確認・修正したうえで、単価マスターに基づく見積書をExcelおよびPDFで出力できる状態を目指す。

到達ゴールは以下とする。

- スマートフォンから音声入力し、文字起こし結果を確認・編集できる。
- AI解析により、顧客情報、作業内容、数量、単位、備考、確認不足項目、業者指示事項候補を抽出できる。
- Excelから単価マスターを取り込み、見積明細候補と照合できる。
- 見積編集画面で顧客、件名、明細、数量、単価、税区分、備考、業者指示事項を編集できる。
- 業者指示事項は内部情報として管理し、顧客向けPDFには表示しない。
- 見積書を標準テンプレートのExcelおよびA4縦PDFで出力できる。
- 見積一覧・詳細から過去の見積を確認できる。

## 開発フェーズ

### フェーズ0: MVP仕様確定・技術方針決定

マイルストーン: 開発着手可能な仕様と技術前提が確定している。

- 初期ターゲット業種をリフォーム・外壁塗装系として仮決定する。
- MVPの画面範囲、データ項目、権限範囲を確定する。
- AI API、音声文字起こし方式、Excel/PDF生成方式を選定する。
- 標準見積書Excelテンプレートと単価マスターExcelテンプレートを定義する。
- 音声データとAI解析履歴の保存方針を決める。

### フェーズ1: 基盤・マスター管理

マイルストーン: 会社、ユーザー、顧客、単価マスターを登録・参照できる。

- 認証、会社単位のデータ分離、権限管理を実装する。
- 会社情報、顧客マスター、単価マスターのCRUDを実装する。
- 見積、見積明細、AI解析履歴のデータモデルを作成する。
- 見積番号採番、税計算、利益率計算などの共通ロジックを整備する。

### フェーズ2: 単価マスターExcel取込

マイルストーン: 既存Excel単価表から単価マスターを初回登録できる。

- Excelアップロード、列名読取、列マッピングを実装する。
- 候補列の自動推定、取り込みプレビュー、行単位のエラー表示を実装する。
- 必須項目不足、単価の数値不正、重複品目の検出を実装する。
- 追加登録または既存データ上書きの取込方式を実装する。
- 成功件数、失敗件数、警告件数を表示する。

### フェーズ3: 音声入力・AI解析

マイルストーン: 音声またはテキストから見積明細候補を生成できる。

- スマートフォンブラウザ向けの録音開始、停止、再録音UIを実装する。
- 録音音声を文字起こしAPIへ送信し、結果を表示・編集できるようにする。
- テキスト入力からもAI解析を開始できるようにする。
- AI解析プロンプトとJSONスキーマを実装する。
- 顧客情報、現場住所、作業内容、数量、単位、備考、確認事項を抽出する。
- 単価マスターとの候補照合を行い、品目候補と参考金額を提示する。
- 業者指示事項候補を抽出し、見積全体または明細行に紐づけられる形で保持する。
- AI結果は候補として扱い、ユーザー確定前には見積へ反映しない制御を入れる。

### フェーズ4: 見積作成・編集

マイルストーン: AI解析結果または手入力から見積を完成できる。

- 新規見積作成、顧客選択・新規作成、件名、見積日、有効期限、担当者入力を実装する。
- 見積明細行の追加、編集、削除、並び替えを実装する。
- 品目、数量、単位、単価、税区分、備考を編集できるようにする。
- 小計、消費税、合計金額の自動計算を実装する。
- 値引き行、諸経費行を追加できるようにする。
- 見積ステータス、複製、修正見積作成を実装する。
- 業者指示事項を見積全体および明細行ごとに登録・編集できるようにする。
- 顧客向け表示項目と内部向け表示項目を明確に分ける。

### フェーズ5: Excel/PDF出力・一覧詳細

マイルストーン: 確定した見積を出力し、一覧から再確認できる。

- 標準テンプレートによるExcel見積書出力を実装する。
- Excelには会社情報、顧客情報、見積明細、合計金額、備考、支払条件、業者指示事項を反映する。
- 顧客提出用PDF出力を実装する。
- PDFには業者指示事項を表示しない。
- スマートフォンからPDF共有、PCからPDF/Excelダウンロードをできるようにする。
- 見積一覧、検索、絞り込み、詳細表示を実装する。

### フェーズ6: MVP検証・初期リリース準備

マイルストーン: 初期ユーザーに限定公開できる品質に到達している。

- 主要業務フローの結合テストを実施する。
- スマートフォン実機相当で音声入力、AI解析、PDF共有を確認する。
- Excel取込とExcel/PDF出力の帳票レイアウトを確認する。
- AI解析結果の人間確認フロー、業者指示事項の非表示制御を重点確認する。
- セキュリティ、会社別データ分離、権限制御を確認する。

## 優先順位

## 依存関係

| 依存元 | 依存先 | 理由 |
| --- | --- | --- |
| 見積編集 | データモデル、顧客マスター、単価マスター | 見積は顧客と単価を参照して作成するため。 |
| 単価候補照合 | 単価マスター管理、Excel取込 | AI候補を既存単価に紐づけるため。 |
| AI解析結果確認 | AI文字起こし、AI見積解析、単価候補照合 | 候補表示には文字起こし、抽出、単価照合が必要なため。 |
| 音声入力 | AI文字起こしAPI選定 | 録音形式、送信方式、制限時間がAPI仕様に依存するため。 |
| Excel出力 | 見積編集、会社情報、標準Excelテンプレート | 出力項目とレイアウトが確定見積に依存するため。 |
| PDF出力 | 見積編集、会社情報、PDFテンプレート | 顧客提出用の表示項目を見積データから生成するため。 |
| 業者指示事項 | 見積編集、AI見積解析 | 手入力とAI抽出候補の両方から内部指示を扱うため。 |
| 権限制御 | 認証、ユーザー管理 | 管理者/一般担当者で操作可否を分けるため。 |
| 会社別データ分離 | 認証、全データモデル | 他社データ参照防止の前提となるため。 |
| MVP検証 | P0機能一式 | 業務フロー全体で受入確認するため。 |

実装順序は、先にデータ基盤とマスター管理を固め、その後にExcel取込、見積編集、AI解析、出力へ進める。ただし、AIプロンプトと帳票テンプレートは仕様ブレが大きいため、フェーズ1から並行して試作し、フェーズ3以降で本実装へ統合する。

## 初期リリース判定条件

初期リリースは、以下をすべて満たす場合に可とする。

- 管理者が会社情報、ユーザー、顧客、単価マスターを登録・編集できる。
- 既存Excel単価表をアップロードし、列マッピング、プレビュー、エラー確認、取込実行ができる。
- スマートフォンブラウザで音声録音し、文字起こし結果を表示・編集できる。
- テキスト入力からも見積作成を開始できる。
- AI解析で見積明細候補、確認事項、業者指示事項候補を生成できる。
- AI解析結果はユーザーが確認・確定するまで見積に反映されない。
- 単価マスターを優先して単価と参考金額を提示できる。
- 見積編集画面で明細、数量、単価、税区分、備考、業者指示事項を修正できる。
- 業者指示事項が顧客向けPDFに表示されない。
- Excel見積書に業者指示事項を社内確認用または業者指示用として出力できる。
- PDF見積書をスマートフォンから共有、PCからダウンロードできる。
- 見積一覧・詳細で作成済み見積を確認できる。
- 会社別データ分離と基本権限が機能している。
- 音声入力から見積明細候補生成までが通常ケースで1分以内に完了する。
- 100明細以内のExcel/PDF出力がそれぞれ10秒以内を目標として大きく逸脱しない。
- 主要ブラウザの最新版で、スマートフォン優先画面とPC優先画面の基本操作が完了できる。
