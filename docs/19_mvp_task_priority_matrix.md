## 2. MVPの実装方針

MVPでは、建設業の見積作成に必要な最短業務フローを通せる状態を目標とする。

対象範囲は以下とする。

| 区分 | MVPで実装する内容 |
| --- | --- |
| 認証・会社分離 | Supabase Auth、会社別データ分離、RLS、ユーザー権限 |
| マスター管理 | 顧客マスター、単価マスター、単価マスターExcel取込 |
| 見積作成 | 顧客選択、見積ヘッダ入力、見積明細入力、ドラッグ並び替え |
| AI入力 | 音声またはテキストから見積明細候補を生成 |
| AI制御 | 顧客情報などヘッダ項目はAI入力しない。明細のみAI解析対象とする |
| 単価連携 | AIが品目候補を推測し、単価マスターから選択候補を出す。単位はマスターの値を採用する |
| 業者指示事項 | 見積明細ごとに保持。見積書PDFには出さず、Excel出力には含める |
| 帳票出力 | 見積書Excel出力、顧客向けPDF出力 |
| 課金 | Stripeによる契約状態管理、オプション機能フラグ |
| オプション | 打ち合わせ録音機能は別オプションとして設計のみ保持し、MVP本体とは切り離す |

## 3. 優先順位定義

| 優先度 | 意味 | 判断基準 |
| --- | --- | --- |
| P0 | MVP成立に必須 | これがないと見積作成の主要フローが動かない |
| P1 | 初期リリース品質に必要 | なくてもデモは可能だが、実運用や販売時に弱くなる |
| P2 | MVP後でよい | 初期導入後の改善、拡張、個別要望対応 |

## 10. MVP受け入れ条件

MVPは以下を満たした時点で初期リリース可能とする。

| No | 条件 |
| --- | --- |
| 1 | 管理者がログインし、自社データのみ操作できる |
| 2 | 顧客マスターを登録し、見積作成時に選択できる |
| 3 | 単価マスターをExcelから取り込み、品目、単位、単価、外部品目IDを保存できる |
| 4 | 見積ヘッダは手入力・選択で登録でき、AIは入力しない |
| 5 | 音声またはテキストから見積明細候補を生成できる |
| 6 | 単位はAI判断ではなく単価マスターから取得される |
| 7 | AI候補はユーザー確認後にのみ見積へ反映される |
| 8 | 見積明細に場所、品目、数量、単位、単価、金額、備考、業者指示事項を保持できる |
| 9 | 明細をドラッグアンドドロップで並び替えできる |
| 10 | 顧客向けPDFに業者指示事項が出ない |
| 11 | Excel出力には業者指示事項が含まれる |
| 12 | Stripe契約状態により利用可否を制御できる |
| 13 | 他会社の顧客、単価、見積、ファイルを参照できない |
| 14 | 主要フローのE2Eテストが通る |

## 11. QA観点

| 観点 | チェック内容 |
| --- | --- |
| 会社分離 | tenant_idを差し替えても他会社データへアクセスできない |
| 権限 | memberが管理者専用機能を操作できない |
| AI制御 | 顧客情報、見積日、有効期限、担当者をAIが上書きしない |
| 単価照合 | AIが単位を推測せず、単価マスターの単位を使う |
| 帳票差分 | PDFには業者指示事項なし、Excelにはあり |
| Excel取込 | 必須列不足、単価不正、重複、外部品目ID重複を検出できる |
| 並び替え | 明細順序が保存後も維持される |
| 課金 | 契約停止時に保護対象機能を使えない |
| 音声 | 文字起こし失敗時に再試行・手入力へ戻れる |
| 性能 | 100明細程度の見積で編集・出力が大きく遅延しない |

## 12. 最初に着手するタスク

次の開発着手順を推奨する。

1. P0-01 Supabase環境確認
2. P0-02 認証基盤実装
3. P0-03 会社別データ分離実装
4. P0-06 顧客マスターCRUD
5. P0-07 単価マスターCRUD
6. P0-08 単価マスターExcel取込
7. P0-09 見積ヘッダ作成・編集
8. P0-10 見積明細CRUD

理由は、AI機能や帳票出力がすべて顧客、単価、見積データに依存するためである。先にDBとマスターを固めると、UI、AI、帳票担当を並列に動かしやすくなる。

## 13. 開発時の注意事項

- `tenant_id` はクライアントから受け取らず、必ずログインセッションから決定する。
- 顧客情報、見積ヘッダ情報はAIで自動入力しない。
- 見積明細の単位はAIで判断せず、単価マスターに紐づく単位を使用する。
- AI結果は候補として扱い、ユーザー確定前に見積へ反映しない。
- 業者指示事項は内部情報として扱い、PDF見積書には出さない。
- Excel出力には業者指示事項を含める。
- 単価マスターには外部品目IDを持たせ、将来の他システム連携に備える。
- 打ち合わせ録音機能は別オプションとして扱い、MVP本体の必須条件にしない。
