# 画面サイズ設計書

## 1. 目的

本ドキュメントは、音声AI見積作成システムの画面サイズ、対応端末、ブレイクポイント、主要画面ごとのレイアウト方針を定義する。

本システムは、現場担当者がスマートフォンで音声入力し、事務担当者または管理者がPCで見積確認・マスター管理を行う利用形態を前提とする。

## 2. 基本方針

- スマートフォン、タブレット、PCブラウザに対応する。
- 現場入力はスマートフォンを最優先する。
- 見積編集、一覧検索、Excel取り込み、マスター管理はPCでの操作性を重視する。
- すべての画面はレスポンシブ対応とする。
- スマートフォンでは横スクロールを極力発生させない。
- 明細表やマスター一覧など横幅が必要な情報は、スマートフォンではカード表示または簡略表示に切り替える。

## 3. 対応端末

| 端末区分 | 主な利用者 | 主な用途 |
| --- | --- | --- |
| スマートフォン | 現場担当者、営業担当者 | 音声入力、AI解析確認、見積の簡易修正、PDF共有 |
| タブレット | 現場担当者、営業担当者 | 現場での見積確認、明細修正、顧客説明 |
| PC | 事務担当者、管理者、経営者 | 見積編集、Excel出力、単価マスター管理、顧客管理、設定 |

## 4. 推奨画面サイズ

### 4.1 スマートフォン

| 用途 | 幅 | 高さ | 備考 |
| --- | --- | --- | --- |
| 最小対応 | 360px | 640px | 小型Android相当 |
| 標準想定 | 390px | 844px | iPhone標準サイズ相当 |
| 大型想定 | 430px | 932px | 大型スマートフォン相当 |

### 4.2 タブレット

| 用途 | 幅 | 高さ | 備考 |
| --- | --- | --- | --- |
| 縦向き | 768px | 1024px | iPad縦向き相当 |
| 横向き | 1024px | 768px | iPad横向き相当 |

### 4.3 PC

| 用途 | 幅 | 高さ | 備考 |
| --- | --- | --- | --- |
| 最小PC | 1280px | 720px | ノートPC最小想定 |
| 標準PC | 1440px | 900px | 業務利用の標準想定 |
| 大画面 | 1920px | 1080px | 事務所PC、外部モニター想定 |

## 5. ブレイクポイント

| 区分 | 幅 | レイアウト方針 |
| --- | --- | --- |
| Mobile | 0px - 767px | 1カラム、下部固定アクション、カード型明細 |
| Tablet | 768px - 1023px | 1から2カラム、主要操作を画面下または右側に配置 |
| Desktop | 1024px - 1439px | 左サイドバー + メイン領域 |
| Wide Desktop | 1440px以上 | 左サイドバー + メイン領域 + 右サイドパネル |

## 6. 共通レイアウト仕様

### 6.1 スマートフォン

- 画面幅は100%を使用する。
- 左右余白は16pxを基本とする。
- 主要ボタンは画面下部に固定表示する。
- タップ領域は44px以上を確保する。
- 入力項目は1列表示とする。
- 明細行はテーブルではなくカード形式を基本とする。
- 音声入力ボタンは片手操作しやすい位置に配置する。
- 業者指示事項は「内部用・PDF非表示」と明示する。

### 6.2 タブレット

- 入力フォームは1から2列表示とする。
- 明細は簡易テーブルまたはカード形式を画面幅に応じて切り替える。
- AI解析結果と見積明細を上下または左右に分けて表示できるようにする。

### 6.3 PC

- 左サイドバー幅は240pxを基本とする。
- メイン領域の最大幅は画面用途に応じて設定する。
- 一覧画面はテーブル表示を基本とする。
- 見積編集画面では、明細テーブルと右サイドパネルを併用する。
- 右サイドパネルにはAI解析元、確認事項、業者指示事項、出力設定などを表示する。

## 7. 主要画面別サイズ方針

### 7.1 音声入力画面

| 端末 | 方針 |
| --- | --- |
| スマートフォン | 最優先。録音ボタン、録音状態、文字起こし結果を縦に配置する。 |
| タブレット | 音声入力と文字起こし結果を上下または左右に配置する。 |
| PC | テキスト入力併用画面として扱う。音声入力は補助的に配置する。 |

打ち合わせ録音は別オプションとし、MVP本体の音声入力画面は見積明細作成用の短い録音を対象にする。

### 7.2 AI解析結果確認画面

| 端末 | 方針 |
| --- | --- |
| スマートフォン | 抽出結果、確認事項、明細候補、業者指示事項候補をセクション分けする。 |
| タブレット | 抽出結果と明細候補を2カラム表示できるようにする。 |
| PC | 左に解析結果、中央に明細候補、右に確認事項を配置する。 |

### 7.3 見積編集画面

| 端末 | 方針 |
| --- | --- |
| スマートフォン | 基本情報、明細、備考、業者指示事項をタブまたはアコーディオンで分割する。 |
| タブレット | 明細編集と補足情報を2カラムで表示する。 |
| PC | 明細テーブルを中心に、右サイドパネルで業者指示事項や出力設定を表示する。 |

### 7.4 単価マスターExcel取込画面

| 端末 | 方針 |
| --- | --- |
| スマートフォン | 閲覧と簡易確認のみ。大量データの取り込み作業は非推奨とする。 |
| タブレット | 簡易アップロードとプレビュー確認に対応する。 |
| PC | 推奨環境。アップロード、列マッピング、プレビュー、エラー確認を快適に操作できるようにする。 |

### 7.5 見積一覧・顧客一覧・単価マスター一覧

| 端末 | 方針 |
| --- | --- |
| スマートフォン | カード一覧表示。重要項目のみ表示し、詳細画面へ遷移する。 |
| タブレット | 簡易テーブルまたはカード一覧を切り替える。 |
| PC | テーブル表示。検索、絞り込み、並び替え、ページングを表示する。 |

## 8. 明細表示仕様

### 8.1 スマートフォン

見積明細はカード形式を基本とする。

表示項目:

- 場所
- 品目名
- 数量
- 単位
- 単価
- 金額
- 備考
- 業者指示事項

業者指示事項には「内部用・PDF非表示」のラベルを表示する。

### 8.2 PC

見積明細はテーブル形式を基本とする。

推奨列:

- 行番号
- 場所
- 品目名
- 数量
- 単位
- 単価
- 金額
- 備考
- 業者指示事項

横幅が不足する場合は、業者指示事項を右サイドパネルまたは展開行で編集できるようにする。

## 9. 出力画面の表示方針

出力操作では、帳票種別ごとの表示項目差分を明確にする。

| 出力種別 | 用途 | 業者指示事項 |
| --- | --- | --- |
| 顧客提出用PDF | 顧客へ提出 | 出力しない |
| 顧客提出用Excel | 顧客へ提出、または編集用 | 原則出力しない |
| 社内確認用Excel | 社内確認 | 出力できる |
| 業者指示用Excel | 協力業者への指示 | 出力できる |

出力ダイアログでは、選択中の帳票に業者指示事項が含まれるかを明示する。

## 10. UI検証用画面サイズ

MVPでは、以下の画面サイズで表示確認を行う。

| 種別 | 幅 | 高さ |
| --- | --- | --- |
| Small Mobile | 360px | 640px |
| Standard Mobile | 390px | 844px |
| Large Mobile | 430px | 932px |
| Tablet Portrait | 768px | 1024px |
| Tablet Landscape | 1024px | 768px |
| Desktop | 1280px | 720px |
| Standard Desktop | 1440px | 900px |
| Wide Desktop | 1920px | 1080px |

## 11. 重要な注意点

- 顧客向け備考と業者指示事項は、画面上で明確に分離する。
- 業者指示事項はPDF非表示であることを、編集画面、プレビュー画面、出力画面で一貫して表示する。
- スマートフォンでは、見積明細の大量編集を主用途にしない。
- Excel取り込みはPC推奨とし、スマートフォンでは無理に全機能を提供しない。
- 主要操作ボタンは、どの画面サイズでも画面外に隠れないようにする。
- テキストがボタンやカードからはみ出さないようにする。
