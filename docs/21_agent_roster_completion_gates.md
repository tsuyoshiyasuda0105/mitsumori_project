# エージェント一覧・完了条件管理表

## 1. 目的

本書は、音声AI見積作成システムを並列で進めるために、現在定義済みのエージェントを一覧化し、各エージェントが「作業したつもり」で終わらないようにする完了条件を定義する。

運用上の基本方針は以下とする。

| 方針 | 内容 |
| --- | --- |
| 並列化 | DB、API、UI、AI、帳票、マーケ、QAを分けて同時に進める |
| 分離 | 同じファイルを複数エージェントが同時編集しない |
| 完了ゲート | 各エージェントは成果物、検証結果、残課題を出すまで完了できない |
| QA独立 | 実装担当とは別にQA/バグチェックエージェントが確認する |
| フック活用 | 作業開始、編集後、完了前に自動チェックを入れる |

## 3. サボり防止の共通完了条件

すべてのエージェントは、以下を満たさない限り完了扱いにしない。

| No | 共通完了条件 | 判定方法 |
| ---: | --- | --- |
| 1 | 何を作ったかが明記されている | 成果物、変更ファイル、対象画面、対象資料を記載 |
| 2 | 何を確認したかが明記されている | 実行コマンド、確認画面、比較表、レビュー観点を記載 |
| 3 | 仕様との対応が明記されている | 参照したdocs、要件、設計項目を記載 |
| 4 | 未対応・残課題が明記されている | 残課題がない場合も「なし」と明記 |
| 5 | 次の担当に渡せる状態である | 入力、出力、依存、引き継ぎ事項を記載 |
| 6 | 重大リスクを自己点検している | 情報漏えい、金額誤り、AI誤反映、帳票漏えいを確認 |
| 7 | QAエージェントに渡す材料がある | 再現手順、テストデータ、確認観点を記載 |

## 4. エージェント別の完了ゲート

### 4.1 開発系エージェント

| エージェント | 完了条件 | 完了不可条件 |
| --- | --- | --- |
| DB/RLSエージェント | migration SQL、RLS policy、tenant分離テストSQL、主要テーブル定義が揃っている | RLS未設定、`tenant_id`をクライアント入力前提にしている、他社データ遮断テストがない |
| APIエージェント | 入出力schema、認証確認、tenant決定ロジック、エラー処理、APIテスト観点が揃っている | 未ログインで操作可能、他社ID指定を拒否しない、validationがない |
| UI/UXエージェント | PC/スマホの主要導線、表示崩れ確認、フォーム状態、エラー状態、保存後再表示が確認済み | スマホ未確認、長文や明細が崩れる、保存成功/失敗状態がない |
| AI音声エージェント | AI JSON schema、ヘッダ非入力、単位非推論、単価マスター候補照合、ユーザー確定前非反映が確認済み | AIが顧客/日付/担当者を作る、AIが単位を決める、未確認で明細反映する |
| Excel/PDFエージェント | Excel取込、Excel出力、PDF出力、業者指示事項の出力差分、破損チェックが確認済み | PDFに業者指示事項が出る、Excelに業者指示事項が出ない、金額不一致 |
| Stripeエージェント | Checkout、Webhook、契約状態保存、プラン別機能制御、オプション判定が確認済み | Webhook署名未検証、契約停止でも機能が使える、録音オプション分離なし |

### 4.2 QA/セキュリティ系エージェント

| エージェント | 完了条件 | 完了不可条件 |
| --- | --- | --- |
| QA/バグチェックエージェント | Critical/High観点、再現手順、期待結果、実際結果、重大度、修正案を出している | 「問題なさそう」だけで終わる、再現手順がない、セキュリティ観点がない |
| セキュリティ確認担当 | 他社データ参照、権限突破、署名URL、ファイル、AI履歴、録音データの確認がある | RLSだけ見てAPIやStorageを見ていない |
| 回帰確認担当 | 修正箇所と周辺機能の再テスト結果を出している | 修正箇所だけ見て帳票/AI/金額への副作用を見ていない |

### 4.3 マーケティング系エージェント

| エージェント | 完了条件 | 完了不可条件 |
| --- | --- | --- |
| マーケットエージェント | 競合、勝ち筋、差別化、ターゲット、価格仮説、LP反映案が揃っている | 競合名だけ、抽象的な差別化だけ、実装/訴求に落ちていない |
| LP改善エージェント | title、description、canonical、OGP、FAQ、CTA、構造化データ、表示確認が済んでいる | CTAがない、SEOメタ未確認、スマホ表示未確認 |
| SEO/ブログエージェント | タイトル、slug、H2、画像、表、FAQ、内部リンク、CTA、meta descriptionが揃っている | 画像なし、表なし、FAQなし、LP誘導なし、禁止表現が残る |
| SNSエージェント | 媒体別投稿文、目的、ハッシュタグ、LP誘導、投稿カレンダーがある | 投稿文だけ、ターゲットや目的がない |
| 動画エージェント | 台本、カット割り、テロップ、ナレーション、サムネ案、生成AI用promptがある | アイデアだけ、撮影/生成に渡せない |
| 営業エージェント | デモ台本、ヒアリング項目、反論対応、初回メール、導入チェックリストがある | 営業資料だけで商談導線がない |
| 顧客対応/CSエージェント | FAQ、返信テンプレ、初回導入手順、既存Excel確認手順がある | 問い合わせ返信だけで導入手順がない |
| 運用/ドキュメントエージェント | 更新資料、変更履歴、次タスク、未決事項、担当引き継ぎが整理されている | 資料を作っただけで最新化ルールがない |

## 6. 並列作業時の衝突防止ルール

| ルール | 内容 |
| --- | --- |
| ファイル担当を分ける | DB担当はSQL、UI担当はcomponents、SEO担当はblog/LPなど担当範囲を分ける |
| 同一ファイル編集は統括が担当 | 複数エージェントの成果を同じファイルへ反映する場合は統括が統合する |
| QAは編集しない前提で始める | QAエージェントは原則レビューと指摘を先に出し、修正は別タスク化する |
| マーケと開発を分離 | LP/ブログ/SNSはMVP実装と並列で進める |
| 完了報告を標準化 | すべてのエージェントは「成果物、確認結果、残課題、次アクション」で報告する |

## 7. 完了報告テンプレート

各エージェントは、作業終了時に以下の形式で報告する。

```text
## 成果物
- 作成/変更したもの:
- 変更ファイル:

## 参照した仕様
- 参照資料:
- 対応した要件:

## 確認結果
- 実行したチェック:
- 確認できたこと:

## リスク確認
- 会社別データ分離:
- AI制御:
- 帳票漏えい:
- 金額計算:

## 残課題
- 未対応:
- 判断待ち:

## 次アクション
- 次に渡す担当:
- 次にやること:
```

## 8. フックで強制するチェック

Claude Code hooksを使う場合、以下のゲートを設定する。

| Hook | 目的 | 推奨チェック |
| --- | --- | --- |
| `SessionStart` | 作業開始時に前提を読み込ませる | 要件、DB設計、API設計、QA計画、エージェント一覧の参照 |
| `PreToolUse` | 危険操作を止める | `.env`表示、破壊的git操作、本番secret出力、無断削除のブロック |
| `PostToolUse` | 編集後に未検証状態を検知する | 変更ファイル一覧、対象領域判定、必要チェックの提示 |
| `Stop` | 完了前にサボりを止める | 成果物、確認結果、残課題、次アクションがない完了報告を差し戻す |
| `Notification` | 放置や承認待ちを見逃さない | 承認待ち、長時間停止、QA待ちの通知 |

## 9. Stop Hookで止めるべき完了報告

以下のような報告は完了扱いにしない。

| NG報告 | 理由 |
| --- | --- |
| 「実装しました」だけ | 変更ファイル、検証結果、残課題がない |
| 「問題ありません」だけ | QA観点、再現手順、確認範囲がない |
| 「SEO対応しました」だけ | title、description、canonical、OGP、構造化データの確認がない |
| 「ブログを書きました」だけ | 画像、表、FAQ、CTA、内部リンクの確認がない |
| 「DBを作りました」だけ | RLS、tenant分離、テストSQLの確認がない |
| 「PDF出力できました」だけ | 業者指示事項が出ていない確認がない |
| 「AI候補を作りました」だけ | ヘッダ非入力、単位非推論、ユーザー確定前非反映の確認がない |

## 12. 結論

このプロジェクトでは、エージェントを単なる担当名ではなく、完了条件つきの作業単位として運用する。

特に重要なのは以下である。

- DB/RLSは、他社データ遮断テストなしでは完了不可
- AI音声は、ヘッダ非入力、単位非推論、未確認反映なしを満たさないと完了不可
- Excel/PDFは、PDF非表示、Excel表示の業者指示事項チェックなしでは完了不可
- SEO/ブログは、CTA、内部リンク、FAQ、画像、表なしでは完了不可
- QAは、再現手順と修正案なしでは完了不可
- すべてのエージェントは、成果物、確認結果、残課題、次アクションなしでは完了不可

この条件をhookとQAゲートで強制すれば、各エージェントの作業品質を一定以上に保てる。
