# 業務ドメイン設計書

## 想定業務フロー

### 1. 現場ヒアリング

現場担当者は、スマートフォンから音声またはテキストで顧客要望、現場状況、工事範囲、数量の目安、希望時期、注意点を入力する。

入力時点では、顧客へ提出する見積情報と、社内・協力業者向けの作業指示情報が混在しやすいため、AI解析後の確認画面で以下に分離する。

| 区分 | 内容 | 顧客向け帳票への表示 |
| --- | --- | --- |
| 顧客向け見積情報 | 件名、場所、品目、数量、単位、単価、金額、備考、支払条件 | 表示する |
| 社内確認情報 | 値引き理由、担当者メモ、確認不足項目 | 表示しない |
| 業者指示事項 | 養生範囲、搬入経路、施工順、現場注意、協力業者への依頼内容 | PDF見積書には表示しない。必要時のみ業者指示用Excelへ出力する |

### 2. AI解析と見積候補生成

AIは文字起こし結果から、顧客情報、現場住所、場所、品目、数量、単位、備考、確認不足項目、業者指示事項候補を抽出する。AIの出力は確定データではなく、ユーザー確認前の候補として扱う。

金額算出は、AIが推測した単価ではなく、原則として会社ごとの単価マスターを優先する。単価マスターは「品目、単位、単価」が紐づいた基準データとし、AIは音声内容から該当しそうな品目を推測する。担当者は提示された品目候補から採用する品目を選択し、選択された品目の単位と単価を見積明細へ反映する。

### 3. 見積編集

担当者または事務担当者は、見積編集画面で以下を確認する。

- 顧客名、現場住所、件名、見積日、有効期限
- 見積明細の場所、品目、数量、単位、単価、金額、備考、業者指示事項
- 値引き、諸経費、端数調整の有無
- 顧客に見せる備考と、内部向けメモの分離
- 業者指示事項が顧客向け帳票に混入していないこと

### 4. 出力と共有

顧客提出用としてPDFまたはExcel見積書を出力する。顧客提出用PDFには、会社情報、顧客情報、見積明細、合計金額、顧客向け備考のみを表示する。

社内確認用または業者指示用Excelを出力する場合のみ、業者指示事項、社内メモを含められる設計とする。出力種別ごとに表示項目を明確に分ける。

### 5. 修正見積

顧客から変更依頼があった場合は、既存見積を複製して修正見積を作成する。修正理由、変更前後の差分、社内メモは内部情報として保持し、顧客向け帳票には必要な変更後内容のみを表示する。

### 金額計算ルール

- 明細金額は原則として `数量 x 単価` で計算する。
- 数量または単価を空欄にできる「一式」行を許可するが、金額は担当者確認を必須とする。
- 値引きは通常明細と分け、値引き行として負数で管理する。
- 諸経費、現場管理費、廃材処分費は通常明細として管理し、必要に応じてテンプレートから追加する。
- 社内メモは顧客向け帳票に出力しない。

## 単価マスター項目

単価マスターは、見積作成時の品目選択、単位初期値、単価初期値、AI照合の基準として扱う。会社ごとに分離し、他社データを参照できないようにする。

| 項目 | 内容 | 必須 | 備考 |
| --- | --- | --- | --- |
| 品目ID | システム内部ID | 必須 | 自動採番 |
| 会社ID | テナント分離用 | 必須 | 会社ごとに管理 |
| 品目 | 見積明細に表示する品目名 | 必須 | 例: 足場設置、高圧洗浄、下塗り |
| 単位 | m2、m、箇所、式、日、人工など | 必須 | Excel取込時に正規化 |
| 単価 | 見積明細に初期反映する単価 | 必須 | 税抜基準を推奨 |
| 有効フラグ | 見積候補に出すか | 必須 | 無効品目は過去見積参照用に保持 |

### 単価マスター運用

- AI照合では、音声の作業内容、言い換え、類似表現から単価マスターの品目候補を提示する。
- ユーザーが品目候補を選択すると、単位と単価を見積明細へ初期反映する。
- 単価は初期値として扱い、担当者が見積編集画面で修正できる。
- 単価マスターに一致しない場合は、手入力品目として明細を作成し、単位と単価の入力を求める。
- 無効化した品目は新規見積候補から除外するが、過去見積の再表示には使用できるように履歴を残す。

## 業者指示事項の扱い

業者指示事項は、社内または協力業者向けの内部情報として管理する。顧客向けPDF見積書には表示しない。

### 登録単位

| 登録単位 | 用途 | 例 |
| --- | --- | --- |
| 見積全体 | 案件全体に関する注意 | 駐車スペースが狭いため軽バン推奨 |
| 明細行 | 特定作業に対する指示 | 北面のみ下地劣化が強いためケレン増し |
| 単価マスター | 品目ごとの標準指示 | 高圧洗浄後は十分乾燥させる |
| AI解析履歴 | 音声から抽出された未確定候補 | 隣家側の養生に注意との発話あり |

### 表示・出力ルール

- 顧客提出用PDFには出力しない。
- 顧客提出用Excelにも原則出力しない。
- 社内確認用Excelでは、明細ごとの業者指示事項を出力できる。
- 業者指示用Excelでは、必要に応じて顧客向け金額を非表示にし、場所、品目、数量、単位、業者指示事項を中心に出力する。
- 業者指示事項を顧客向け備考へ転記する場合は、担当者が明示的に操作する。

### AI抽出時の注意

- AIが抽出した業者指示事項は候補として表示し、ユーザー確定前に保存済み指示として扱わない。
- 「お客様に説明すること」と「職人に伝えること」は別項目として分類する。
- 危険作業、近隣配慮、追加費用に関わる事項は確認不足項目としても表示する。

### テンプレート共通方針

- 初期テンプレートは、数量拾いしやすい品目と、一式管理が現実的な品目を混在させる。
- 品目名は顧客が読んでも理解しやすい名称を基本とする。
- AIが数量を抽出できない場合に備え、確認不足項目として「施工面積」「施工範囲」「材料グレード」「既存状態」を出す。
- 外壁塗装では m2、m、式、箇所を中心にする。
- リフォームでは 式、台、箇所、m2、人工を中心にする。

## Excel取込仕様の注意点

### 取込対象

初期取込は単価マスターを対象とする。見積書そのものの取込や過去見積の移行は将来拡張とし、MVPでは対象外または別設計とする。

### 必須列と推奨列

| 区分 | 列名例 | 対応項目 |
| --- | --- | --- |
| 必須 | 品目名、項目名、工事項目 | 品目 |
| 必須 | 単位 | 単位 |
| 必須 | 単価、標準単価、見積単価 | 単価 |
| 推奨 | 備考、摘要 | 標準顧客向け備考 |
| 推奨 | 指示事項、業者メモ、施工メモ | 標準業者指示 |

### 列マッピング

- Excelの列名は会社ごとに異なるため、列名候補を自動推定し、ユーザーがプレビュー画面で修正できるようにする。
- 「備考」と「業者メモ」が混在している場合は、顧客向け備考に取り込むか、標準業者指示に取り込むかを明示的に選択させる。
- 業者指示事項は内部情報として扱い、品目や備考へ自動混入させない。

### データ検証

- 品目、単位、単価が不足している行はエラーとする。
- 単価は数値として解釈できない場合にエラーとする。
- カンマ、円記号、税込表記が含まれる金額は正規化して取り込む。
- 単位表記は `㎡`、`m2`、`平米` などを同一単位として正規化できるようにする。
- 重複判定は、品目と単位の組み合わせで判定する。
- 空行、集計行、小計行、見出し行は取り込み対象外としてプレビューで除外候補にする。

### 取り込み方式

- 取り込み前にプレビューを必須とする。
- ユーザーは取り込み対象行を選択できる。
- 既存データとの重複時は、上書き、追加、スキップを選択できる。
- 取り込み結果として、成功件数、失敗件数、警告件数を表示する。
- 失敗行は、行番号、原因、修正例を表示する。

### セキュリティと情報分離

- Excel取込時も会社IDを必ず付与し、他社の単価マスターと混在させない。
- 取り込んだ業者指示事項は内部情報として分類する。
- 顧客向け帳票に出力される可能性がある列は、プレビュー画面で明示する。
- AI照合用キーワードに社外秘情報が含まれる場合があるため、顧客向け出力項目とは別管理にする。
