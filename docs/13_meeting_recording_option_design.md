# 打ち合わせ録音オプション設計

## 1. 方針

顧客との打ち合わせ録音機能は、MVP本体には含めず、別オプションとして提供する。

MVP本体では、見積明細を作成するための短い音声入力を優先する。打ち合わせ録音は、営業支援・顧客対応履歴・AI要約まで含む追加価値として切り出す。

## 2. 本体機能との違い

| 区分 | MVP本体 | 打ち合わせ録音オプション |
| --- | --- | --- |
| 主目的 | 見積明細を素早く作成する | 顧客との会話を営業メモとして残す |
| 録音時間 | 短時間の明細入力を想定 | 長めの打ち合わせ録音を想定 |
| 出力 | 見積明細候補 | 文字起こし、要約、顧客要望、確認事項、見積反映候補 |
| 反映先 | 見積明細 | 営業メモ、顧客履歴、見積反映候補 |
| 契約 | 標準機能 | 別オプション |

## 3. オプションで提供する機能

- 顧客との打ち合わせ録音
- 録音開始前の同意確認
- 録音の一時保存、停止、再録音
- 録音ファイル保存
- 文字起こし
- 要点整理
- 顧客要望の抽出
- 確認事項の抽出
- 見積反映候補の抽出
- 顧客履歴、見積履歴への紐づけ
- ユーザー確認後の見積への反映

## 4. 非表示・利用制御

オプション無効の会社では、以下を表示または利用できないようにする。

- ダッシュボードの打ち合わせ録音ショートカット
- 見積編集画面の打ち合わせ録音ボタン
- 顧客詳細画面の打ち合わせ録音ボタン
- 打ち合わせ録音一覧、詳細画面
- `/api/meeting-recordings` 系API

API側では、画面表示の有無に関わらず、契約プランまたは機能フラグを確認する。

## 5. 権限・プラン管理

会社単位で以下のような機能フラグを持つ。

| フラグ | 意味 |
| --- | --- |
| `meeting_recording_enabled` | 打ち合わせ録音機能を利用できる |
| `meeting_summary_enabled` | 打ち合わせ要約AIを利用できる |
| `meeting_apply_enabled` | 打ち合わせ内容から見積反映候補を作成できる |

初期は `meeting_recording_enabled` の1つにまとめてもよい。将来、録音のみ、AI要約付き、見積反映付きのようにプランを分けられる設計にする。

## 6. 課金設計の考え方

打ち合わせ録音は、AI利用量と保存容量が増えやすいため、標準機能とは分けて管理する。

課金候補:

- 月額オプション料金
- 利用ユーザー数ごとの追加料金
- 録音時間または文字起こし時間の上限
- 保存容量の上限
- 超過分従量課金

MVP初期では、課金ロジックを細かく作り込みすぎず、会社単位の有効/無効フラグと利用量ログを残すところから始める。

課金基盤はStripe Billingを利用する。打ち合わせ録音オプションはStripeの追加商品または価格として管理し、Webhookで契約状態をSupabase Postgresへ同期する。アプリ内ではStripe APIを都度参照せず、同期済みの `tenants.feature_flags.meeting_recording_enabled` と契約状態を参照して機能を出し分ける。

Stripe連携の基本ルール:

- 会社単位でStripe Customerを作成する。
- 打ち合わせ録音オプションの契約状態はWebhookでDBへ同期する。
- `invoice.payment_failed` や `customer.subscription.deleted` を受けた場合は、管理者通知、猶予期間、機能停止の順に処理する。
- 録音時間、文字起こし時間、保存容量は利用量ログとして残し、将来の従量課金に備える。

## 7. UI方針

- オプション無効時は、打ち合わせ録音ボタンを表示しない。
- オプション未契約を示すアップセル表示は、初期は管理者向け設定画面に限定する。
- 現場担当者の見積編集画面には、契約していない機能を目立たせない。
- オプション有効時も、見積明細用の音声入力と打ち合わせ録音を混同しないラベルにする。

推奨ラベル:

- 見積明細用: `明細を音声入力`
- 打ち合わせ用: `打ち合わせを録音`

## 8. セキュリティ・同意

- 録音開始前に、顧客へ録音同意を確認済みであることをチェックする。
- 録音データ、文字起こし、要約、見積反映候補はすべて会社単位で分離する。
- 顧客提出用PDFには、録音内容、営業メモ、AI要約、業者指示事項を出力しない。
- 署名付きURL発行時は、必ず会社IDと権限を検証する。

## 9. MVPへの影響

打ち合わせ録音を別オプションにすることで、MVP本体は以下に集中できる。

- 顧客選択
- 見積明細の音声入力
- AIによる明細候補抽出
- 単価マスター照合
- 見積編集
- Excel/PDF出力

そのため、初期開発の複雑さ、AI利用量、録音同意まわりの運用負荷を抑えられる。
